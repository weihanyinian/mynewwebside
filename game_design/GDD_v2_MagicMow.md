## 📌 设计支柱

| # | 支柱 | 玩家感受 |
|---|------|---------|
| 1 | **爆发感** | 每次技能释放都要有"哇"的反应，屏幕抖动、粒子飞溅 |
| 2 | **成长快感** | 每 30 秒就要感受到明显变强，不能等待 |
| 3 | **选择张力** | 每次升级选技能都要纠结，没有废技能 |
| 4 | **肉鸽惊喜** | 每局都是新体验，组合出意想不到的 Build |
| 5 | **爽感收关** | Boss 死的时候要壮观，不能敷衍 |

## 🔄 核心游戏循环

```
即时循环（0-10秒）
└─ 移动 → 敌人自动聚拢 → 技能自动触发 → 击杀掉落经验宝石
   └─ 拾取宝石 → 数字跳动 → 爽

会话循环（5-20分钟 / 一局）
└─ 升级选技能 → 强化组合 → 关卡波次递增 → Boss战
   └─ 通关 or 死亡 → 解锁元进度 → 再来一局

长期循环（多局累计）
└─ 解锁新角色 → 解锁初始技能 → 挑战更高难度
   └─ 成就系统 → 炫耀 / 分享
```

## 📊 升级系统（1-50级完整设计）

### 经验曲线

```
等级区段    | 升一级需经验 | 设计意图
1-10       | 10~50       | 极速起飞，30秒一级
11-20      | 60~150      | 开始感受成长
21-35      | 200~500     | Build成型，策略加深
36-50      | 600~1500    | 冲顶挑战，每级都是大关
```

### 经验加成来源

| 来源 | 加成量 | 备注 |
|------|-------|------|
| 连击杀敌（10连击） | +15% | 保持进攻手感 |
| 磁铁效果（技能） | +0% 经验但扩大拾取范围 | 让宝石飞来 |
| 经验加成词条 | +10%/+20%/+35% | 三档词条 |

## 🔗 技能搭配协同系统（Synergy System）

### 元素共鸣（火 + 冰 + 雷）

```
当同时激活：🔥火系 + ❄️冰系 + ⚡雷系
→ 触发「元素风暴」：场上所有敌人受到持续元素伤害
  基础伤害 = 火伤 + 冰伤 + 雷伤 的平均值 × 3
  特殊效果：元素风暴中敌人护甲归零，受到伤害+50%
```

### 技能链式反应

```
示例Build：陨石流
D06(雷云) + A05(陨石) + C09(流星雨加持) + E11(雷霆万钧)

触发链：
  陨石落下 → 雷云劈击辅助 → 流星雨加持数量加成 → 击杀触发雷霆万钧落雷
  → 落雷击杀触发连锁闪电 → 连锁闪电击杀触发蝴蝶效应
  = 一套连招清屏效果
```

## ⚖️ 平衡设计

### 伤害公式

```
实际伤害 = 基础伤害 × (1 + 攻击加成%) × (1 + 元素加成%) × 暴击倍率

暴击倍率 = 暴击发生时 2.0（可被技能提升至3.5上限）
```

### 平衡杠杆

| 变量 | 基础值 | 范围 | 说明 |
|------|--------|------|------|
| 玩家HP | 100 | 50-300 | 技能可突破上限 |
| 移动速度 | 160px/s | 80-320 | 不能太快 |
| 技能CD(普通) | 2.5s | 0.5s-8s | 手感测试确认 |
| Boss HP倍率 | ×50 | ×30-×100 | 相对同波次小怪 |
| 暴击率上限 | 60% | - | 防无脑暴击Build |
| 技能伤害加成上限 | +500% | - | 允许很强但有上界 |

```
  └─ 存档同步：增量同步（仅上传差异）
```

### 数据库设计

```sql
-- 用户表
CREATE TABLE user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE,
  password_hash VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 游戏存档表
CREATE TABLE game_save (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT,
  character_type VARCHAR(20),      -- 'mage' / 'swordsman' / 'summoner'
  max_level INT DEFAULT 1,          -- 历史最高等级
  total_kills INT DEFAULT 0,        -- 历史总击杀
  total_games INT DEFAULT 0,       -- 总游戏次数
  total_wins INT DEFAULT 0,        -- 通关次数
  unlocked_characters VARCHAR(255), -- 已解锁角色 JSON
  unlocked_skills VARCHAR(1000),   -- 已解锁技能树 JSON
  skill_points INT DEFAULT 0,      -- 剩余技能点
  achievements JSON,               -- 成就进度 JSON
  best_scores JSON,                -- 每角色最高分 JSON
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES user(id)
);

-- 技能树表（独立存储，便于扩展）
CREATE TABLE skill_tree (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT,
  skill_id VARCHAR(10),             -- 'A01', 'B02' 等
  skill_level INT DEFAULT 0,        -- 0-3
  FOREIGN KEY (user_id) REFERENCES user(id),
  UNIQUE KEY (user_id, skill_id)
);
```

### API 设计

```
认证相关：
  POST   /api/auth/register        注册
  POST   /api/auth/login           登录，返回JWT
  GET    /api/auth/me              获取当前用户

存档相关：
  GET    /api/game/save            获取存档
  PUT    /api/game/save            保存存档（增量）
  POST   /api/game/save/partial    增量更新（如只更新分数）

元数据：
  GET    /api/game/skills          获取技能树定义（纯配置）
  GET    /api/game/config          获取游戏配置（波次、敌人参数等）
```
